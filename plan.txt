-------  Введение ----------------

Всем, добрый вечер, представляю результат работы над учебным проектом,
целью которого было создание фронтенд-части интернет-магазина по продаже товаров для животных.

Приложение создано на основе фреймворка РЕАКТ.
Также в приложении использованы дополнительные библиотеки:

        "axios": "^1.10.0" -                для http  запросов на бекенд,

        "redux": "^5.0.1",
        "@reduxjs/toolkit": "^2.8.2",
        "react-redux": "^9.2.0",
        "redux-persist": "^6.0.0",          для управления глобальным состоянием приложения, дублирование в ЛОКАЛ СТОРЕДЖ

        "react-hook-form": "^7.60.0",
        "@hookform/resolvers": "^5.1.1",
        "yup": "^1.6.1"                     для управления и валидации форм
        
        "react-router-dom": "^7.7.0",       для навигации по приложению
        
        "react-select": "^5.10.2",          компонент-аналог html тегу SELECT с возможностью стилизации - применяется в панели фильтров и сортировки 
        
Приложение доступно по адресу     https://pet-shop-frontend-blue.vercel.app/

-------- проблемы с проектом ----------
    1. ControlBar - интеграция с "react-hook-form" - не используется тег form, получение данных с помощью watch
    2. SingleProductPage (Breadcrumbs) - организация связанных запросов, пытался решить с помощью отдельных хуков useFetch - работало не стабильно
                            -  решение - последовательные await запросы.   В новом бекенде - проблема отпала
    3. API - передача в запрос нескольких параметров (пагинация+фильры) - решено ДЕСТРУКТУРИЗАЦИЯ
